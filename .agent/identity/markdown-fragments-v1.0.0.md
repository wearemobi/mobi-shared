## 1. Overview
This document defines the official Markdown fragments (Header and Footer) for all **M.O.B.I.™** documentation. These fragments ensure consistent branding, metadata traceability, and professional presentation across the Grand Fleet.

## 2. Unified Header
Use this fragment at the very beginning of every Markdown document. Replace `[Title]` with the actual title of the document.

### Source Code:
```markdown
# <img src="https://wearemobi.com/icon-light.svg" width="24" height="24" valign="middle"> M.O.B.I.™ · [Title]
```

## 4. Usage Rules
- **Header:** Use the standard Markdown H1 (`#`) with the official inline icon.
- **Footer:** The footer should be preceded by a horizontal rule (`---`).
- **Metadata:** The hidden HTML comment in the footer is used by M.O.B.I. agents for document verification and version tracking. Do not remove it.
