## Context & Goal
Enhance the `MobiFooter` component to support application-specific metadata (name and version) and decouple layout margins from the `MobiLogoHero` primitive to allow for more flexible composition in different UI contexts.

## Changelog

### Changed
- **`MobiFooter`** — added `appName` and `version` props to display application metadata in the footer.
- **`MobiLogoHero`** — removed hardcoded `mb-8` margin from the component to improve composability. Consumers are now responsible for vertical spacing.
- **`Documentation`** — updated `MobiFooter.md` to document the new props.
- **`Example`** — updated `example/App.tsx` to showcase the parametric footer with library metadata.

## Files Modified
| File | Action |
|---|---|
| `src/components/MobiFooter.tsx` | Updated — Added `appName` and `version` support |

| `docs/api/MobiFooter.md` | Updated docs |

## Acceptance Criteria
