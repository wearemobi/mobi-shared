# <img src="https://wearemobi.com/icon-light.svg" width="24" height="24" valign="middle"> M.O.B.I.™ · Fleet Integration v1.3.1

## Context & Goal
Establish standardized tactical interfaces for the M.O.B.I.™ Grand Fleet microservices. This version integrates **Mobi-Vault** for asset storage and **Mobi-Reactor** for energy accounting.

## Changelog

### Added
- **Mobi-Vault Tactical Client** — Introduced `useMobiVault` for multipart file ingestion.
- **Automated Vault Ingestion** — Enhanced `MobiDropbox` with direct `vaultUrl` support.
- **Mobi-Reactor Tactical Client** — Introduced `useMobiReactor` for energy reporting and querying.
- **Sovereign Fleet Types** — Standardized interfaces for `FileMetadata`, `ConsumeRequest`, and `QueryResponse`.

## API Specification

### 1. Asset Storage (Mobi-Vault)
`MobiDropbox` now accepts a `vaultUrl` to handle uploads internally via `useMobiVault`.

```tsx
<MobiDropbox vaultUrl="http://localhost:8081" />
```

### 2. Energy Accounting (Mobi-Reactor)
`useMobiReactor` handles Haki consumption and history.

```typescript
const { consume, query } = useMobiReactor();
```
