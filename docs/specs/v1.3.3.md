## Context & Goal
Introduce support for the **Gemini Backup Cloud Core** directly within the M.O.B.I.‚ĄĘ Chat architecture. This ensures immediate, high-availability out-of-the-box operation across all deployed modules (bypassing local sandboxes when sovereign local Ollama nodes are unreachable or in production). 

To maintain the compact, premium design footprint of the `MobiChatInput` text area, the **Engine Switcher is visually hidden from the user toolbar by default**, and the internal engine state **defaults to `gemini`** (running completely transparently). However, the full parameter structure (`activeEngine`, `onEngineChange`) is **fully preserved in components and hooks**, enabling client applications to programmatically toggle and control the active engine context as required.

## API Specification

### 1. Gemini Backup Core (`useMobiGemini`)
Provides a standard, stateless `chat` function to communicate directly with Google's cloud APIs using a development-level key.

```typescript
const { chat, isProcessing, lastResponse } = useMobiGemini();

// Example Usage
const handleSend = async () => {
  const result = await chat("Analyze fleet status...");
  console.log(result.response); // Direct from Gemini with Mobi persona instructions
};
```

### 2. Multi-Engine Unified Send (`onSendMessage`)
The callback passed into `MobiChatWidget` can now route queries depending on the active engine.

```typescript
const handleSendMessage = async (message: string, model: string, engine: 'mobi' | 'gemini') => {
  if (engine === 'gemini') {
    // Route to Gemini
  } else {
    // Route to Sovereign Local Agentic
  }
};
```
