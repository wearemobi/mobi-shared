# Specification v1.4.3 - Mobi-Edge Health Check, Fullscreen Mode & Explicit Error Handling

## Overview
This specification details the comprehensive features introduced in **M.O.B.I.™ Shared v1.4.3**. It upgrades `useMobiEdge`, `MobiChatWidget`, and `MobiChatEdge` to introduce a secure offline health flow, custom popout/fullscreen capabilities, detailed and explicit error reporting with markdown JSON rendering, fully realized message actions (copy, share, retry), and robust horizontal stacking configurations to support multiple side-by-side chat sentinels.

---

## 1. Client-Side Health Definition

### Architectural Objective
Prevent downstream console errors and request cascades when the Mobi Edge service is offline by performing a lightweight, unauthorized health check. If backend status is not `'online'`, the client disables catalog and status queries, immediately entering an offline/unconnected state.

---

## 2. Fullscreen & Maximize Capabilities

### Fullscreen State Toggle
Introduce a new full-screen capability to expand the chat panel to fill the entire viewport. This is triggered by a new popout icon button in the header right next to the close `X` button.

### Visual State Transition
* **Expanded (Fullscreen)**: Container stretches to `fixed inset-0 z-[9999] flex flex-col`. The chat window scales to `w-full h-full border-0 rounded-none shadow-none`.
* **Compact (Default)**: Container scales back to its standard dimensions `w-[380px]` (or custom offsets) in the bottom-right corner.
* **Auto-Reset on Close**: Closing the chat window automatically resets the fullscreen state to `false`, ensuring that re-opening the chat starts in the compact view, and preventing any full-screen transparent overlays from blocking click events on the main page.

---

## 3. Explicit Error Reporting & Haki Check Simplification

### API Error Body Parsing
When the `/api/v1/agentic/infer` API fails (non-2xx response), the client no longer throws a generic `'Inference Failure'` error. Instead, it reads the error body (either a detailed JSON or raw response text) and packages it:
```typescript
throw new Error(`Inference Failure\n\n\`\`\`json\n${errorDetails}\n\`\`\``);
```
Since message contents are parsed via `MobiMarkdown` (GitHub Flavored Markdown), this displays the raw JSON error payload directly inside the chat window with syntax highlighting and perfect readability.

### Haki Check Removal
Removed hardcoded `403` status check simulation which mapped to a generic `INSUFFICIENT HAKI` message. This allows authentic 403/401 error payloads returned from the backend (like catalog or tier authorization failures) to be rendered explicitly.

---

## 4. Message Actions: Copy, Share & Retry

### Copy Action
Copies the message text to the user's clipboard and triggers a sleek, glassmorphic toast notification banner overlaying the header (`COPIED TO CLIPBOARD`).

### Share Action
Uses the web standard `navigator.share` if supported (for seamless mobile integration) and falls back to copying the text and showing a toast notification if unsupported.

### Robust Retry Action
* **Original Bug**: Clicking "Retry" sent the string of the error message back to the LLM.
* **Resolution**: Clicking "Retry" parses backwards from the failed message index to find the nearest `user` role prompt. It re-triggers `sendMessage(prompt, model)` using the exact same prompt and original model configurations.
* Also triggers a toast notification (`RETRYING GENERATION`) to provide visual confirmation.

---

## 5. Horizontal Stacking & Overlap Fixes

To support applications displaying multiple floating chat sentinels simultaneously (e.g., global Mobi Edge Agent and ACME Sentinel), positioning has been completely decoupled from hardcoded coordinates.

### Additions to `MobiChatWidgetProps`
Exposed the `containerClassName` property to allow custom positioning of `MobiChatWidget`:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `containerClassName` | `string` | `undefined` | Custom styling class to override the default bottom-right floating position. |

### Positioning Rules
The custom positioning class (`containerClassName`) is now applied **both when open and when closed**, preventing closed trigger icons from collapsing into the same default corner (`bottom-6 right-6`).

---

## 6. API Specifications

### New Data Types (`MobiEdgeHealth`)
```typescript
export interface MobiEdgeHealth {
  status: 'online' | string;
  service: string;
  version: string;
  environment: string;
  fingerprint: string;
}
```

### Hook API Additions (`useMobiEdge`)
The returned object from the `useMobiEdge` hook is expanded to expose the following fields:

| Prop | Type | Description |
|------|------|-------------|
| `health` | `MobiEdgeHealth \| null` | The parsed health payload returned by the service, or `null` if unreachable/unhealthy. |
| `isHealthy` | `boolean` | Fast boolean indicator confirming if the endpoint has an active, `'online'` status. |
| `refreshHealth` | `() => Promise<boolean>` | A manual trigger to re-check the health of the endpoint. |
