# Specification v1.3.6: Form System & System Hardening (Audit)

## Overview
This specification covers the introduction of the M.O.B.I.‚Ñ¢ Form Architecture alongside an exhaustive safety, accessibility, and architectural audit/remediation across 16 legacy components and hooks.

---

## Changelog

### Added (Form Architecture)
- **MobiInput** ‚Äî [NEW] Unified text, email, number, money, password, and textarea input element. Supports floating states, prefix/suffix icons, currency formatting, toggleable password visibility, error borders, and full ARIA description mappings.
- **MobiDropdown** ‚Äî [NEW] Accessible custom dropdown menu element support single selection from option maps with dynamic options rendering and validation bindings.
- **MobiFormLabel** ‚Äî [NEW] Title, section, and standard field label headings adhering to M.O.B.I.‚Ñ¢ styling guidelines.
- **FormError** ‚Äî [NEW] Standalone inline validation feedback element with red alert accent styling.
- **useMobiForm** ‚Äî [NEW] Advanced forms manager hook managing `values`, validation `errors`, `touched` states, async `isSubmitting` tracking, and form-level reset functions.
- **mobiValidators** ‚Äî [NEW] Schema validator utility suite (`required`, `email`, `minLength`, `maxLength`, `min`, `max`, `number`, `currency`, `pattern`).

### Changed / Remediated (System Audit)
- **MobiAlert** ‚Äî Fixed stale closure in `useEffect`, wrapped `handleClose` in `useCallback` defined before references, added `role="alert"`, `aria-live`, and marked SVGs with `aria-hidden="true"`.
- **MobiButton** ‚Äî Added `loading` indicator spinner, `suffixIcon` slot, `aria-busy` attribute, and managed disabled states when loading.
- **MobiCard** ‚Äî Extends `HTMLAttributes<HTMLDivElement>`, and fixed bug where footer wrapper would always render even if empty.
- **MobiErrorBoundary** ‚Äî Added `onError` production logging hook, stored `errorInfo` in state, and exposed interactive nested component stack fallback.
- **MobiProgress** ‚Äî Added `role="progressbar"`, `aria-valuenow`, `aria-valuemin`, and `aria-valuemax` parameters.
- **MobiSwitcher** ‚Äî Exported `MobiSwitcherProps` interface, integrated `role="group"`, and applied `aria-pressed` / `aria-label` attributes to active states.
- **MobiSentinelMenu** ‚Äî Added keyboard `Escape` handler, integrated `role="menu"` / `role="menuitem"`, and resolved TypeScript compiler typing errors.
- **MobiDropbox** ‚Äî Fixed relative positioning wrapper issue, added `onInvalidFiles` notification payload, and resolved race condition awaiting Vault microservice result before calling parent success triggers.
- **useMobiAuth** ‚Äî Strongly typed login state substituting `any` reference with strict `MobiUser` structure.
- **useMobiTheme** ‚Äî Wrapped setter in `useCallback`, strongly-typed event listeners, and simplified CSS `.dark` class toggles.
- **useMobiClipboard** ‚Äî Prevented memory leaks by storing timer instances inside a React `useRef` hook for explicit unmount cleanup.
- **useMobiEnergy** ‚Äî Converted `isLow` state flag into a derived inline constant, removing unnecessary dependency re-renders.
- **useMobiVault** ‚Äî Removed hardcoded sandbox environment configurations and added fallback checks if `baseUrl` is absent.
- **useMobiChat** ‚Äî Enforced high-uniqueness message IDs via `crypto.randomUUID()` and stabilized callback dependencies using closures.

## API Specification

### `useMobiForm`
```typescript
const form = useMobiForm({
  initialValues: { email: '', age: 0 },
  validationRules: {
    email: [mobiValidators.required(), mobiValidators.email()],
    age: [mobiValidators.required(), mobiValidators.min(18)],
  },
  onSubmit: async (values) => {
    await submitToServer(values);
  }
});
```

### `MobiInput`
```tsx
<MobiInput
  type="email"
  label="Admin Address"
  value={email}
  onChange={handleChange}
  error={touched.email ? errors.email : undefined}
  required
/>
```
