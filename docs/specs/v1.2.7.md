## Context & Goal
Enhance the **Asset Ingestion Pipeline** by evolving the `MobiDropbox` component. This release focuses on improving accessibility for end-users and increasing operational throughput by enabling parallel file processing.

## Changelog

### Added
- **Multi-File Ingestion** — `MobiDropbox` now supports the `multiple` property, enabling users to select and process multiple digital assets in a single interaction.
- **Direct Interaction Vector** — Integrated a hidden file picker triggered by a container click. This provides a secondary interaction path for users who prefer standard file exploration over drag-and-drop.

### Changed
- **`MobiDropbox` API** — Added `multiple` (boolean) prop to control selection behavior.
- **Interaction Logic** — Container now supports `onClick` event mapped to a technical file input reference.

### Fixed
- **Input Sanitization** — Automatically resets the file input after each ingestion cycle to allow re-selection of the same assets.

## Files Modified

## Acceptance Criteria
- [x] `MobiDropbox` allows selecting multiple files when `multiple={true}`.
- [x] Clicking anywhere in the drop zone opens the OS file explorer.
- [x] Drag-and-drop continues to function correctly for single and multiple files.
- [x] Extension filtering (`acceptedExtensions`) is enforced on both interaction vectors (Drag and Click).
