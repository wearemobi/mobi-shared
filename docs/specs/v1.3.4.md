# <img src="https://wearemobi.com/icon-light.svg" width="24" height="24" valign="middle"> M.O.B.I.™ · Slim Sentinel & Reusable Theme Switcher v1.3.4

## Context & Goal
Streamline the M.O.B.I.™ sovereign header profile and drop-down sentinel components for a highly minimalistic and ultra-premium visual presence. This is accomplished by stripping redundant label contexts, dropping secondary switches, and refactoring the primary appearance toggle into its own reusable, theme-aware core component.

## Changelog

### Changed
- **MobiUserBadge** — Redesigned the `'condensed'` trigger button layout to render as a perfect `h-12 w-12` circular avatar container with no plan text or padding offsets, preserving a clean "C" circular presence on the page toolbar.
- **MobiSentinelMenu** — Streamlined dropdown settings section:
  - Removed the "Idioma" (Language) row and all internal language-switching states completely.
  - Removed the "Interfaz" text label, centering the theme toggle switcher row inside the config block.
  - Refactored the inline switcher markup into the newly extracted reusable `<MobiThemeSwitcher />`.

### Added
- **MobiThemeSwitcher** — [NEW] A highly robust, standalone component containing the light, auto, and dark svg options. It operates in both controlled (props-driven) and uncontrolled/automatic modes (utilizing `useMobiTheme()` internally when no `theme` is provided).

## API Specification

### `MobiThemeSwitcher`
Can be instantiated standalone anywhere in the host application to provide independent theme controls:

```tsx
import { MobiThemeSwitcher } from '@wearemobi/shared';

// Uncontrolled (Binds automatically to useMobiTheme hook)
<MobiThemeSwitcher />

// Controlled (Binds to custom host states)
<MobiThemeSwitcher 
  theme="dark" 
  onChange={(nextTheme) => console.log(nextTheme)} 
  className="my-custom-layout-class"
/>
```
