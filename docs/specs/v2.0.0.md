# Specification v2.0.0 - Mobi UI Primitives & Premium Aesthetics

## Overview
This specification details the transition to the **M.O.B.I.™ Shared v2.0.0** architecture. The release completely redesigns the component library to focus on premium, dynamic aesthetics utilizing modern frontend standards.

---

## 1. Major Redesigns

- **MobiBentoGrid**: Introduced a supercharged bento grid with premium layouts.
- **MobiTimeline**: Added right ghosted rail connector and refined spacing for historical sequences.
- **MobiThemeToggle**: Replaced old theme switcher with a more elegant toggle and `MobiThemeProvider`.
- **MobiButton / MobiBadge**: Implemented uniform CVA tokenization and modern chip-like tracking.

## 2. New Component Additions

- **MobiCodeBlock**: Syntax highlighting with Shiki compatibility and literal newline formatting.
- **MobiEnergyMeter**: High-fidelity animated meter for agentic consumption.
- **MobiSearchCommand**: Adds a robust searchable command palette integration.
- **MobiTabs & Segmented Control**: High-fidelity animated segmented views.
- **MobiListView**: New density-aware vertical list views.

## 3. Architecture Adjustments
- Cleaned up legacy CSS files in favor of inline absolute definitions to prevent tailwind class-bleeding from host applications (specifically targeting `MobiFooter` and `MobiSidebar`).
- Deprecated legacy components moving them to `/scratch`.
- Added new hooks: `useAutoDismiss` and `useMobiToast`.
