# <img src="https://wearemobi.com/icon-light.svg" width="24" height="24" valign="middle"> M.O.B.I.™ · The Universal UI Shield v1.2.0

## Context & Goal
Transform `mobi-shared` from an identity fragment repository into the official, universal UI & Logic library for the entire M.O.B.I. ecosystem. This release cycle stabilizes the "Holy Trinity" of core components and introduces the foundational "Agentic Chat" and "Ingestion" modules.

## Major Milestones (v1.2.x Cycle)

### 1. The Holy Trinity (Universal Core)
- **`<MobiLogo />`**: Reactive SVG implementation of the minimalist 2x2 grid logo.
- **`<MobiFooter />`**: Standardized branding footer with dynamic metadata support.
- **`<MobiSentinelMenu />`**: Premium user dropdown with "Micro" variant support for condensed interfaces.

### 2. Agentic Chat System
- **`<MobiChatWidget />`**: The primary AI assistance popup interface.
- **`<MobiChatFeed />`**: Industrial-grade message stream with Inter/Outfit typography.
- **`<MobiChatInput />`**: Command center with integrated model switching and status indicators.
- **`<MobiEnergyMeter />`**: Visual battery/quota indicator for agentic resources.

### 3. Ingestion Pipeline
- **`<MobiDropbox />`**: Multi-file drag-and-drop container with progress visualization.
- **`<MobiCard />`**: Universal asset container for ingestion results and metadata.
- **`<MobiProgress />`**: Tactical progress bar for asynchronous operations.

### 4. Aesthetics & DNA
- **Soft Industrial Design**: Square geometry, consistent rounding, and iridescent accents.
- **Typography**: Unified Inter (System) and Outfit (Content) font stacks.
- **Theme Sync**: Global CSS variable synchronization for seamless integration into host apps.

### 5. Logic & Hooks
- **`useMobiAuth`**: Foundational hook for JS Bridge v1.19 integration.
- **`useMobiChat`**: State management for agentic conversations.
- **`useMobiEnergy`**: Interface for resource consumption tracking.

## Technical Implementation
- **Vite Library Mode**: Compiled as UMD and ESM modules for universal consumption.
- **Tailwind v4 Integration**: Surgical CSS detection and zero-clash styling.
- **I18n Readiness**: Parameterized labels across all UI components.

## Acceptance Criteria
- [x] Library is buildable and distributable via NPM/GitHub Registry.
- [x] All core components are visually premium and functionally resilient.
- [x] Auth and Chat logic is abstracted into reusable hooks.
