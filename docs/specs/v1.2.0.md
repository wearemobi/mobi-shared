## Context & Goal
Transform `mobi-shared` into the official, universal UI & Logic library for the entire M.O.B.I. ecosystem. The goal is to provide a single, powerful source of truth for the "Holy Trinity" of components and auth logic, making them easily importable via `@mobi/shared` in any project (Chassis, wearemobi.com, etc.).

## Implementation Plan

### 1. Library Core Foundation
- [ ] Initialize `package.json` as `@mobi/shared`.
- [ ] Setup TypeScript + Vite (Library Mode) for distribution.
- [ ] Define the export strategy (Main Entry Point).

### 2. The Holy Trinity (Universal Components)
- [ ] **`<MobiLogo />`**: Implement the minimalist 2x2 grid logo as a scalable, reactive SVG.
- [ ] **`<MobiFooter />`**: Build the standardized footer with dynamic metadata and branding.
- [ ] **`<MobiSentinelMenu />`**: Create the premium user dropdown (Login/Register/Profile) with modern aesthetics.

### 3. The Nervous System (Hooks & Logic)
- [ ] **`useMobiAuth`**: Develop the foundational hook for **JS Bridge v1.19** integration.
- [ ] Support for `app_id` configuration and OAuth-ready state management.

### 4. DX & Simplicity
- [ ] Document the "Sweet Path" for universal usage: `import { ... } from '@mobi/shared'`.
- [ ] Ensure CSS variables are encapsulated and don't clash with host applications.

## Acceptance Criteria (Definition of Done)
- [ ] `mobi-shared` is buildable as a distributable package.
- [ ] All three "Holy Trinity" components are functional and visually premium.
- [ ] `useMobiAuth` correctly exposes the auth interface.
- [ ] Simplicity achieved: One import to rule them all.
