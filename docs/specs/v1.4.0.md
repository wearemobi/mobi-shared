# Specification v1.4.0 - Edge API v1 Integration

## Overview
This specification defines the transition of the `mobi-shared` library to use the formalized `v1` API of the MobiEdge platform. This update ensures compatibility with the latest Agentic Memory, Reactor transactional logic, and Sentinel security protocols.

## Changes

### 1. Endpoint Normalization
All API calls from `mobi-shared` (specifically within `useMobiEdge` hook) MUST now use the `/api/v1` prefix.

| Feature | Legacy Path (v2.0.8) | New Path (v1.0.0) |
|---------|---------------------|-------------------|
| Status  | `/api/reactor/status` | `/api/v1/reactor/status` |
| Catalog | `/api/reactor/catalog` | `/api/v1/reactor/catalog` |
| Infer   | `/api/agentic/infer` | `/api/v1/agentic/infer` |
| History | N/A | `/api/v1/agentic/history` |

### 2. Hook Enhancements (`useMobiEdge`)
- **Stateful Conversations**: Added support for `conversationId` to track persistent memory sessions.
- **Session Persistence**: The hook will optionally persist `conversationId` in `localStorage` to maintain context across page refreshes.
- **Agent Targeting**: Added support for `agentId` (default: `mobi-core`) to utilize specific Sentinel personas.
- **Memory Consumption**: Automatically manages `useMemory` flag based on tenant Tier.
- **Dynamic Suggestions**: Returns `suggestions` provided by the agent response.

### 3. Component Updates (`MobiChatEdge`)
- **New Props**:
    - `agentId?: string`: Target specific intelligence units.
    - `persistSession?: boolean`: Whether to save conversation state in local storage.
- **Dynamic UI**:
    - Uses suggestions returned from the API response to populate quick-reply pills.
    - Displays "Memory Active" indicator for PRO/ULTRA tiers.

## Technical Details

### Inference Request
```json
{
  "prompt": "string",
  "model": "string (optional)",
  "agentId": "string (optional)",
  "conversationId": "string (optional)",
  "useMemory": "boolean (optional)"
}
```

### Inference Response
```json
{
  "agent_id": "string",
  "conversation_id": "string",
  "model_used": "string",
  "response": "string",
  "suggestions": ["string"]
}
```

## Backward Compatibility
The hook will attempt to fall back to `baseUrl` without `/api/v1` if a 404 is encountered, ensuring a smooth rollout during the transition phase.
