# Specification v1.3.9: MOBI Edge Reactor Industrialization

## Overview
This specification details the production-ready "industrialization" of the MOBI Edge Reactor integration. It refines the v2.0.8 (Radar) architecture, introducing specialized intelligence selection components, modularized energy telemetry, and robust data-driven categorization for AI models.

---

## Changelog (Industrialization Update)

### Added
- **MobiIntelligenceSelector** — [NEW] Reusable atomic component for model selection.
  * `truncate` + `whitespace-nowrap` logic to prevent UI wrapping.
  * Supports custom labels following the `TIER (slug)` pattern.
  * Integrated tactical status indicator.
- **Toggleable Energy Footer** — Modularized energy monitoring.
  * Detailed telemetry (`haki_used / haki_limit`) is now hidden by default.
  * Interactive expansion triggered by clicking the battery icon in the footer.

### Improved
- **useMobiEdge (Hook Refinement)**:
  * **Sovereign Filtering**: Implemented `resource_kind === 'AI_MODEL'` as the primary discriminator.
  * **BASIC-first Strategy**: Automatic auto-selection of the "BASIC" tier during catalog synchronization.
  * **Schema Alignment**: Renamed `available_resources` to `intelligence_catalog` to match Radar v2.0.8 spec.
  * **Robust Error Handling**: Standardized generic user-facing errors while preserving technical details in console logs.
- **MobiChatEdge (Component Refinement)**:
  * Rebranded as "MobiEdge Agent" with standard tactical welcome message.
  * Re-integrated the iconic Robot Sentinel icon on the chat trigger.
  * Delegated model selection and energy monitoring to modular sub-components.

## API Specification

### `useMobiEdge` (v1.3.9)
```typescript
const { 
  messages, 
  models, 
  status, 
  isProcessing, 
  activeModelId,
  sendMessage 
} = useMobiEdge({
  token: '...',
  tenantId: 'MOBI',
  initialModelId: 'mobi-assist-1b-3b' // Optional: Force initial selection
});
```

### `MobiIntelligenceSelector`
```tsx
<MobiIntelligenceSelector 
  options={models}
  activeId={activeModelId}
  onChange={(id) => setActiveModelId(id)}
  disabled={isProcessing}
/>
```

## Technical Requirements (Radar v2.0.8)

### Data Contract Update
- **Catalog**: `GET /api/reactor/catalog` returns `intelligence_catalog`.
- **Model Object**:
  ```typescript
  {
    slug: string,
    resource_kind: 'AI_MODEL' | 'STORAGE',
    engine_name: 'BASIC' | 'PRO' | 'ULTRA',
    haki_cost: number,
    ...
  }
  ```

### Error Policy
- **403 (Insufficient Haki)**: System-level "RECHARGE REQUIRED" state.
- **500/General Failures**: User-facing generic tactical failure message.
