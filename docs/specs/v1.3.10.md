# Specification v1.3.10: Edge Chat Customization & White-Labeling

## Overview
This specification expands the `MobiChatEdge` component API to allow deep functional and visual customization, enabling host applications (like Novus Honduras) to define the initial state, suggested interactions, and custom branding tokens.

---

## New API Parameters (MobiChatEdgeProps)

| Prop | Type | Description |
|---|---|---|
| `initialMessages` | `MobiEdgeMessage[]` | Overrides the default welcome message with a custom sequence. |
| `suggestions` | `string[]` | (NEW) List of interactive labels (pills) shown above the input for quick actions. |
| `isInitiallyOpen` | `boolean` | (NEW) If true, the chat widget starts in expanded mode. |
| `onSendMessage` | `(msg: string) => void` | (NEW) Callback triggered when the user sends a message. |
| `hideRobotIcon` | `boolean` | (NEW) Option to hide the Sentinel Robot icon on the trigger button. |
| `customWelcome` | `string` | Shortcut to override only the first assistant message content. |

---

## Technical Enhancements

### 1. Suggested Questions (Pills)
The UI will now render a horizontal scrollable list of "pills" above the input area if the `suggestions` prop is provided. Clicking a suggestion automatically triggers `sendMessage`.

### 2. State Management
The `isOpen` state will prioritize:
1. `isOpen` (controlled prop)
2. `isInitiallyOpen` (initial internal state)
3. `false` (default)
