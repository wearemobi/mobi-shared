# Specification v1.4.2 - Collapsible Navigation, Card Variants & Bento Grid

## Overview
This specification details the upgrades introduced in **M.O.B.I.™ Shared v1.4.2**. It enhances hybrid layout flexibility by introducing collapsible desktop sidebar capability in `MobiNav`, dedicated footer actions, color-curated theme variants in `MobiCard`, and an industrialized, reusable `MobiBentoGrid` system.

---

## 1. Collapsible Desktop Navigation Layout

### Architectural Objective
Provide an optional toggle capability for the `MobiNav` desktop sidebar, allowing the viewport to collapse the menu into a clean overlay state identical to the mobile experience.

### API Specifications

#### `MobiNavProps` Additions
| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `collapsible` | `boolean` | No | Allows the desktop sidebar to collapse and toggle using the top-bar trigger (Default: `false`). |
| `footerActions` | `React.ReactNode` | No | Extra custom action slot placed in the sidebar footer layout. |
| `onToggle` | `(isOpen: boolean) => void` | No | Callback fired when the navigation sidebar collapses or expands. |

#### `MobiSidebarProps` Additions
| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `collapsible` | `boolean` | No | Enables toggling behavior on all viewports, disabling the fixed `lg:translate-x-0` lock. |
| `footerActions` | `React.ReactNode` | No | Actions element injected directly above or below the version details. |

## 3. Curated Card Color Variants

`MobiCard` supports a unified color palette matching M.O.B.I.™ branding guidelines:

- **`default`**: Crisp white/charcoal responsive layout surface with default boundaries.
- **`primary`**: Deep tactical slate highlight boundary accents.
- **`accent`**: Subtle tech-blue tint overlay (`bg-blue-500/5`) for informative containers.
- **`success`**: Emerald-hued container boundary (`bg-emerald-500/5`) for active/approved items.
- **`warning`**: Amber-hued warning boundary (`bg-amber-500/5`) for pending states or attention flags.
- **`danger`**: Red-hued danger boundary (`bg-red-500/5`) for alarm matrices.
- **`tactical`**: Ultra-sober deep-dark layout card (`bg-slate-900` / `bg-zinc-950`) designed for dashboard commands.

## 4. Reusable Bento Grid Layout

A structural Bento Grid component providing high visual quality layout alignment for dashboard blocks.

### API Definition

#### `MobiBentoGridProps`
| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `children` | `React.ReactNode` | Yes | Bento item blocks. |
| `columns` | `number` | No | Total grid columns. Maps responsive grid classes (Default: `4`). |
| `className` | `string` | No | Layout overrides. |

#### `MobiBentoItemProps`
| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `children` | `React.ReactNode` | Yes | Content block inside the Bento item. |
| `colSpan` | `1 \| 2 \| 3 \| 4 \| 'full'` | No | Column span footprint of this item block (Default: `1`). |
| `rowSpan` | `1 \| 2 \| 3 \| 'full'` | No | Row span footprint of this item block (Default: `1`). |
| `className` | `string` | No | Additional classes. |
