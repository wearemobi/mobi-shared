# <img src="https://wearemobi.com/icon-light.svg" width="24" height="24" valign="middle"> M.O.B.I.™ · Fleet Sandbox Integration v1.3.2

## Context & Goal
Align the M.O.B.I.™ Grand Fleet microservice clients with the local Kubernetes (k8s) sandbox environment and introduce the **Mobi-Agentic** integration. This ensures developers have a unified, zero-configuration local sandbox experience across Vault, Reactor, and Agentic services.

## Changelog

### Changed
- **Mobi-Vault Tactical Client** — Updated default `baseUrl` to point to the local k8s endpoint: `http://vault.engine.sandbox.grandfleet.local`.
- **Mobi-Reactor Tactical Client** — Updated default `baseUrl` to point to the local k8s endpoint: `http://reactor.engine.sandbox.grandfleet.local`.

### Added
- **Mobi-Agentic Tactical Client** — Introduced `useMobiAgentic` hook to interact with the sovereign AI engine. Uses `http://agentic.engine.sandbox.grandfleet.local` by default and consumes string-based payloads via `/api/chat`.

## API Specification

### 1. Agentic Engine (Mobi-Agentic)
`useMobiAgentic` provides a `chat` method to send text prompts to the local Ollama instance and returns a `ChatResponse`.

```typescript
const { chat, isProcessing, lastResponse } = useMobiAgentic();

// Example Usage
const handleSend = async () => {
  const result = await chat("Analyze this data structure...");
  console.log(result.response);
};
```
