## Context & Goal
Ensure that the `@wearemobi/shared` library is automatically built when installed directly from its Git repository. This fixes issues where the `dist` folder might be missing or outdated in remote deployments (like Cloudflare Pages) that consume the library via Git URL.

## Changelog

### Added
- **`prepare` script** — added `"prepare": "npm run build"` to `package.json`. This lifecycle script runs automatically after `npm install` (and before `npm publish`), ensuring the library is always compiled for consumers.

### Changed
- **Version Bump** — bumped version from `1.2.3` to `1.2.4` to reflect the installation fix.

## Files Modified
| File | Action |
|---|---|
| `package.json` | Updated — Added `prepare` script and bumped version to `1.2.4` |

## Acceptance Criteria
- [x] `npm install` on a consumer project triggers the build of `mobi-shared`.
- [x] `dist` directory is generated automatically without manual intervention.
- [x] Version is correctly set to `1.2.4`.
