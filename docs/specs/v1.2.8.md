## Context & Goal
Increase the modularity of the **Asset Container Layer**. This release focuses on making the `MobiCard` component more adaptable to different branding and content requirements by decoupling the technical footer from hardcoded values.

## Changelog

### Added
- **Custom Footer Support** — `MobiCard` now accepts a `footer` prop (ReactNode), allowing consumers to inject custom branding, status messages, or metadata.

### Changed
- **Default State** — Removed the hardcoded "MOBI Asset Container" text. The footer area is now clean by default unless a prop is provided.

## Files Modified
| File | Action |
|---|---|
| `src/components/MobiCard.tsx` | Updated — Added `footer` prop and removed hardcoded text. |
| `package.json` | Updated — Bumped version to `v1.2.8`. |
| `docs/specs/v1.2.8.md` | **New** — Technical specification for container flexibility. |

## Acceptance Criteria
- [x] `MobiCard` renders custom content in the footer when provided.
- [x] `MobiCard` does not render default text when no footer prop is passed.
- [x] Technical status dots (pulse) remain active in the footer for system feedback.
