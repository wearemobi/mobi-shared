# Specification v1.4.1 - MobiNav & Identity System

## Overview
This specification defines the `MobiNav` component and the broader **M.O.B.I.™ Identity & Iconography** system. It establishes a sovereign, dependency-free visual language for the Grand Fleet, including centralized iconography, application branding, and standardized documentation layouts.

## Architectural Mandate

### 1. Hybrid Responsive Design
- **Desktop (Laptops/Monitors)**: Displays as a persistent, high-density sidebar with micro-animations.
- **Mobile (Smartphones/Tablets)**: Collapses into a tactical hamburger trigger that opens a full-screen or slide-out module drawer.
- **Breakpoints**: Standardized at `768px` (md) for layout transition.

### 2. Component Structure (`MobiNav`)
The component will be composed of three internal functional layers:
- **Trigger Layer**: The hamburger/close icon toggle for mobile states.
- **Manifest Layer**: The list of navigation items (modules), supporting icons and status badges.
- **Identity Layer**: Displays the active Island/Sector DNA at the bottom of the navigator.

## API Definition

### `MobiNavProps`

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `items` | `NavModule[]` | Yes | Array of navigation module objects. |
| `activeId` | `string` | Yes | ID of the currently active module for highlighting. |
| `onNavigate` | `(id: string) => void` | Yes | Callback triggered upon module selection. |
| `isCollapsed` | `boolean` | No | Initial state for desktop minimized sidebar. |
| `showDNA` | `boolean` | No | Whether to show the Island/Sector metadata (Default: `true`). |
| `className` | `string` | No | Additional styling overrides. |

### `NavModule` Interface
```typescript
interface NavModule {
  id: string;
  label: string;
  icon: React.ReactNode;
  path?: string;
  badge?: string | number;
  disabled?: boolean;
  isExternal?: boolean;
}
```

## UI/UX Specifications

### Interactions
- **Hover States**: Sublte glow effects using `mobi-primary` accents.
- **Transitions**: Smooth `300ms` cubic-bezier transforms for sidebar expansion and drawer slides.
- **Glassmorphism**: Backdrop blur effects (`12px`) for the mobile drawer to maintain visual depth.

### Visual Tokens
- **Background**: `var(--mobi-surface-dark)` with `0.8` opacity for glass effects.
- **Border**: `1px solid var(--mobi-border)`.
- **Text**: `var(--mobi-text-muted)` shifting to `var(--mobi-text)` on active/hover.

## 3. Component Structure (`MobiHamburgerMenu`)
The `MobiHamburgerMenu` provides a contextual action manifest triggered by a tactical hamburger icon.

### `MobiHamburgerMenuProps`

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `items` | `MobiMenuItem[]` | Yes | Array of menu action items. |
| `title` | `string` | No | Optional header title for the dropdown. |
| `className` | `string` | No | Additional styling overrides. |

### `MobiMenuItem` Interface
```typescript
interface MobiMenuItem {
  id: string;
  label: string;
  icon?: React.ReactNode;
  danger?: boolean;
  onClick?: () => void;
}
```

## 4. Identity & Iconography Standards

### MobiIcon Registry
A centralized, high-performance registry of tactical glyphs.
- **Geometric Uniformity**: All icons standardized for `24x24` viewboxes.
- **Dependency-Free**: Pure SVG paths wrapped in React Fragments.
- **Context-Aware**: Uses `currentColor` for dynamic theme integration.

### MobiAppLogo Component
Uniform identity mark for fleet applications.
- **Visual Pattern**: Bold white initial centered on a black square.
- **Dynamic Extraction**: Automatically extracts the first letter from the `label` prop.

### MobiSectionHeader Component
Standardized layout for section introductions.
- **Hierarchy**: Headline Badge -> Title -> Description.
- **Versatility**: Conditionally renders components based on prop presence.
