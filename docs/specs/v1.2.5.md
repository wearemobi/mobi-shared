## Context & Goal
Evolve the M.O.B.I. Shared Library into a professional "Technical/Engineering" design system. The goal is to transition from consumer-grade rounded aesthetics to a high-impact, precision-focused look that aligns with the Core Blueprint v1.0.0.

## Changelog

### Added
- **Technical Design Language** — Formalized the use of "Technical Square" geometry (reduced radii) and monospace typography (`JetBrains Mono`).
- **`MobiNavbar`** — Responsive sticky navigation with glassmorphism and automatic "logo-only" mobile state.
- **`MobiHero`** — Refactored Jumbotron with flexible `logo` slot for custom branding injection.
- **`MobiSidebar`** — Responsive navigation drawer that supports permanent display on desktop and overlay on mobile.
- **`MobiSidebarItem`** — Navigation sub-component for sidebar lists.
- **Interactive Documentation** — Refactored `example/DocsPage.tsx` to include the new sidebar and responsive mobile triggers.

### Changed
- **`MobiButton`** — Implemented `technical` mode (monospace font) and unified all variants to follow the new low-radius geometry.
- **Responsive Logic** — Updated all components to use standard Tailwind v4 breakpoints for consistent cross-device behavior.

### Fixed
- **Navbar Mobile Title** — Fixed a bug where the navbar title would wrap and stack on small screens; it is now correctly hidden on mobile.
- **Docs Layout Overlap** — Resolved a visual conflict where the floating menu button overlapped with component badges on mobile viewports.
- **Sidebar Branding** — Updated sidebar footer to use dynamic versioning and changed label to "API Docs".
- **Hero Synchronization** — Harmonized `MobiHero` typography and spacing with the official Chassis standards.
