# Specification v1.2.5 - Technical Engineering Language

**Status**: Draft
**Date**: 2026-05-01
**Author**: Antigravity (M.O.B.I. Agent)

## 1. Objective
Evolve the M.O.B.I. Shared Library into a "Technical/Engineering" design system. This version formalizes the transition from consumer-grade rounded aesthetics to a professional, high-impact engineering look with square corners and monospace typography options.

## 2. Design Language Protocols
*   **Typography**: 
    *   Primary: "Inter" (Sans-serif) for general UI.
    *   Technical: "JetBrains Mono" for engineering-focused views and data-heavy components.
*   **Border Radius**: 
    *   Standardized reduction of radii to `rounded-md` (max) for a "Technical Square" look.
    *   Avoid large circular rounding (`rounded-full`) except for specific avatar/badge elements.
*   **Visual Style**: 
    *   Heavy use of glassmorphism (`backdrop-blur-md`).
    *   MOBI Surface: `#ffffff` (Light) / `#111111` (Dark).
    *   MOBI Border: `#E2E8F0` (Light) / `#222222` (Dark).

## 3. Component Updates

### 3.1 MobiNavbar
*   **Logic**: Responsive "Logo-only" mode for mobile (< 768px).
*   **Glassmorphism**: Sticky header with blurred surface background.
*   **Slots**: `rightContent` for menu/action injection.

### 3.2 MobiHero
*   **Slot System**: Refactored `showLogo` (boolean) to `logo` (ReactNode) for custom branding (e.g., `MobiLogoHero`).
*   **Architecture**: High-impact Jumbotron with centered alignment and flexible subtitle support.

### 3.3 MobiButton
*   **Variants**: `solid`, `outline`, `ghost`, `danger`.
*   **Technical Mode**: Toggleable `technical` prop that switches font to `JetBrains Mono` and tightens padding.
*   **Geometry**: Square/low-radius corners across all variants.

### 3.4 MobiSidebar
*   **Responsiveness**: Permanent sidebar on `lg+`, overlay drawer with backdrop on mobile.
*   **Sub-components**: `MobiSidebarItem` for consistent navigation lists.

## 4. Engineering Standards
*   **Version Control**: Atomic commits following `feat(v1.2.5): ...` or `fix(v1.2.5): ...`.
*   **Responsiveness**: Mobile-first implementation using Tailwind v4 standard breakpoints.
*   **Documentation**: Every new component must have a `docs/api/[Component].md` file and a catalog entry in `example/DocsPage.tsx`.

## 5. Synchronization
This version aligns with the **M.O.B.I.™ Core Blueprint v1.0.0** and prepares the `@wearemobi/shared` library for consumption by the Grand Fleet (Bridge, Chassis, Sentinel).
