## Context & Goal
Perform a surgical cleanup pass on `@wearemobi/shared` to eliminate dead code, remove hardcoded data, and align all components with a data-driven, fleet-ready architecture. This patch prepares the library for Phase 2 (Primitives Layer) by ensuring the existing surface area is clean, parametric, and free of technical debt.

## Changelog

### Removed
- **`MobiThemeSwitcher`** — component deleted. Theme control is now unified inside `MobiSentinelMenu` via `MobiSwitcher`.
- **`MobiPlanBadge.showBullet`** — dead prop removed. No visual bullet was ever rendered.
- **`MobiAlert` inline `<style>`** — `dangerouslySetInnerHTML` eliminated. Keyframes moved to `styles.css`.
- **Dead imports** — `useMobiTheme` from `MobiSwitcher`, `MobiPlanBadge` from `MobiSentinelMenu`.

### Changed
- **`MobiSentinelMenu`** — fully parametrized. All user data now comes through a `user: MobiSentinelMenuUser` prop. Menu actions are data-driven via `items: MobiSentinelMenuItem[]` with automatic danger-item separation. Language state supports both controlled (`lang` + `onLangChange`) and uncontrolled modes.
- **`MobiSwitcher`** — added `hideLabel` prop for icon-only mode. Removed unused `useMobiTheme` dependency.
- **`example/App.tsx`** — updated to consume the new `MobiSentinelMenu` API. Standalone `MobiThemeSwitcher` removed from header.

### Added
- **`@keyframes mobi-progress`** — centralized in `styles.css` for the `MobiAlert` progress bar.
- **`MobiSentinelMenuUser`** interface — typed contract for user identity data.
- **`MobiSentinelMenuItem`** interface — typed contract for menu actions with `danger` flag support.
